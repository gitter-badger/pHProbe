    Introduction

    Characteristics

    • PH range: 0 - 14
    • Temperature range: -2 - 35C
    • Interface: I2C
    • Current use: ~10mA peak, low idle current ~1mA
    • Voltage range: 3.3 - 5.5V

    D Data (yellow)
    - GND (black)

    Getting Started

    To start developing for the device, you need to install the library. Two are provided, one for the Arduino framework, and the other for python3 on the Raspberry Pi.

    If you are using the Arduino IDE, you can go to the library manager (Sketch / Include Library / Manage Libraries…) and search for pH_Probe. Examples are included.

    If you are using PlatformIO, you can install the library using the library manager (PlatformIO / PlatformIO Home / Libraries) and search for pH Probe. Examples are included.

    On your Raspberry Pi, you can install the library with pip install phprobe. Note, you may run into python 2/3 issues. If you get a module not found error, try installing with pip3 install phprobe (replacing pip3 with the appropriate version if pip3 isn’t found). You can find an example at the library’s GitHub page. To access the library, from ecsalinity.ecsalinity import ecsalinity and ec = ecsalinity(i2c_device).

    The shell Example

    An interactive shell interface is provided with both frameworks and is a quick and easy way to get started using the device. You will find the equivalent commands in the code area to the right when applicable.

    Calibration

    Single Point

    #include <phProbe.h>
pH_Probe pH;

ec.calibrateSingle(7.0);

    from phprobe.phprobe import phprobe

pH = phprobe(3)

pH.calibrateSingle(7.0)

      cal 7.0

    Single point calibration determins a percentage-difference between the measured pH and the pH passed to the calibrateSingle() function. After calling it, the results are saved in the devices’s EEPROM and used automatically.

    Dual Point

    You can use dual point calibration for more accurate results between two predesignated points. Two calibration solutions are required, the low and high values you expect to measure between.

    pH.calibrateProbeLow(4.0);  // after measurement, clean probe and place in high solution
pH.calibrateProbeHigh(7.0);

    pH.calibrateProbeLow(4.0) # after measurement, clean probe and place in high solution
pH.calibrateProbeHigh(7.0)

      low 4.0
  high 7.0
    1. Determine the lowest and highest measurement you expect. For example, the lowest level you might measure would be pH 4.0 and the highest might be pH of 7.0. These points will be referred to as referenceLow and referenceHigh
    2. Put the pH probe in a calibration solution at referenceLow and wait for readings to stabilize, call calibrateProbeLow(). Do the same for referenceHigh by calling ‘calibrateProbeHigh’.
    3. By default, the device does not use dual points. A call to useDualPoint() must be made to enable it each time the device is powered.

    You can also set all four values directly using setDualPointCalibration().

    pH.useDualPoint(false);

    pH.useDualPoint(False)

      dp 1

    Each call to measureEC will use the calibration to adjust the reading. It can be disabled by ec.useDualPoint(false);

    Use

    Temperature Compensation

    byte tempConstant = 25;
pH.setTempConstant(tempConstant);
pH.setTempConstant(0xFF);

    pH.setTempConstant(25);
pH.setTempConstant(0xFF);

      tc 1 25
  tc 0
  tc 1 255

    To set the temperature used for compensation, call setTempConstant() and pass the temperature to use. To use the actual temperature, pass 0xFF (255).

    Measurement Time

    Each individual pH measurement takes 10ms. If the default accuracy is set to 9, a call to measureEC will return in about 90ms. A temperature measurement takes 750ms.
